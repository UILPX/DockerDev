# Web 风格设计规范（项目统一版）

本规范用于后续新增页面/模式时保持一致的视觉与交互体验。

## 1. 设计目标

- **一致性优先**：跨 `games` / `reaction` / `main-site` 风格统一。
- **轻量可维护**：优先原生 CSS + 少量设计 token。
- **可读性**：在深色/蓝色背景上保持高对比度文本。

## 2. 颜色规范

### 主色板

- 主背景蓝：`#3b82f6`
- 深色背景：`#0f172a`
- 成功绿：`#22c55e`
- 警示红：`#dc2626`
- 高亮黄（仅目标/强调，不用于整页背景）：`#facc15`
- 文本白：`#ffffff`

### 使用规则

- 页面基础背景优先使用**蓝色系**（`#3b82f6`）或深色（`#0f172a`）。
- 同类模式页面背景应保持同一基调，避免某个模式出现突兀底色。
- 状态色仅用于瞬时反馈（如等待、成功、失败），不要长期占据主背景。

## 3. 排版规范

- 字体：`system-ui`。
- 标题字号：`clamp(28px, 5.8vw, 54px)`。
- 正文字号：`clamp(12px, 3.2vw, 16px)`。
- 行高建议：`1.4 ~ 1.6`。
- 中英文混排时保持按钮/表格最小可点击尺寸。

## 4. 布局规范

- 页面优先使用居中布局（Flex）。
- 主要交互区域保持固定视觉宽高比例（例如 1280x720 的游戏场景）。
- 顶部导航统一放在左上/两端，保留“返回主站 / 返回模式选择”。

## 5. 组件规范

### 按钮

- 圆角：`6px ~ 8px`
- 内边距：`12px 20px` 以上
- hover：透明度轻微变化（`opacity: .9`）

### 表格

- 必须保证移动端横向可滚动。
- 当前用户高亮行使用半透明白底（如 `rgba(255,255,255,0.12)`）。

### 链接

- 默认白色，hover 下划线。
- 顶部导航链接透明度建议 `0.9`。

## 6. 响应式规范

- 断点建议：`520px`（小屏）与 `768px`（平板）。
- 小屏优先保证：
  - 可点击区域不重叠；
  - 关键文案可读；
  - 表格可滚动。

## 7. 交互与状态规范

- 开始前：蓝底 + 清晰指令文案。
- 进行中：根据状态切换红/绿提示，但要快速回归主色基调。
- 结束后：展示结果、最好成绩、排名，并提供完整榜单入口。

## 8. 可访问性要求

- 文字与背景对比度满足可读性（建议至少 4.5:1）。
- 所有关键按钮/链接可通过键盘聚焦。
- 非桌面限制场景应给出明确提示，不得静默失败。

## 9. 新页面落地检查清单

新增页面前请逐项确认：

1. 使用统一主色与字体。
2. 顶部导航入口完整。
3. 结果/错误提示文案双语策略一致（若适用）。
4. 移动端表现可用（或明确限制并给提示）。
5. 与现有模式页面的背景基调一致。
